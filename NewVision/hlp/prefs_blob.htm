Blob Model

Blobs are areas in the image deemed by the tracker to be corresponding to moving objects. Through the process of head candidate selection each blob is assigned one or more head candidates which are points at the top of the blobs representing a possible location of a person's head. Each head candidate has a corresponding foot candidate which is the location on the floor directly below the head.

This dialog controls how blobs are created for your tracking project.

1. Blob minimum area

Blobs that occupy less than this percent of area of the image are discarded. For example: 0.01 = 1%. Increase this value to reduce the number of noise blobs.
Default setting:0.001

2. Blob maximum area

Blobs that occupy more than this percent of area of the image are discarded. For example: 0.2 = 20%. Typically such large blobs can appear if there was a global change in the scene, such as one of the lights shutting off.
Default setting:0.3

3. Max bodies in blob

One blobs can potentially correspond to more than one person. This value sets the upper limit of bodies in a single blob. Increase it for more crowded scenes.
Default setting:3

4. Min ray size

Minimum height of head candidates to be considered as the percentage of the height of the tallest head candidate.
Default setting:0.6
